Заметки по Qt/QtCreator/C/C++/Computer Science:

Фреймворк - Qt. Среда разработки QtCreator

Ко всем объектам Qt осуществляется доступ через переменную ui.

У Сигнала и Слота должны быть одинаковые параметры.

Закладки	Ctrl+M - добавить/удалить закладку
		Alt+M  - список закладок

Когда dll успешно инжиктируется в программу - это значит, что dll успешно нашла точку доступа к основной программе. 
dll всегда подключаются к определеным точкам доступа.
Инжиктирование - внедрениее dll в поток работы логики программы.

Для кнопки есть встроенный в qt метод getEnable (возможно написано неправильно) - при каждом нажатии на кнопку меняется внутреннее состояние кнопки - булевый элемент.

Рекоммендации по работе с QVector - после завершения работы с QVector - очистить, а потом удалить как объект.
Если просто воспользоваться методом clear(), то будет утечка памяти.
Как только закончили работать с объектом - вызывай деструктор. И если дальше этот объект не нужен, то удаить его.

Оперативная память в Windows начинается не с 0, а с 1.
Когда указатель привели к 0, то это означает, что указатель ни к чему не привязан. Привели его к какому-то адресу - то есть мы переопределили указатель.

Библиотека DLL - можно ли отлаживать пошагово? Можно, но если библиотека линкуется статически. Динамические библиотеки отлаживать теоритечески можно, но сложно, так как нужно настроить отладчик и точку доступа. Программа для запуска - это точка доступа.

Создаёшь QCombobox как обычно и привязываешь его к нужной ячейке QTableWidget с помощью метода setCellWidget:

QComboBox *comboBox = new QComboBox();
ui->tableWidget->setCellWidget(0, 0, comboBox); // помещаем в ячейку 0, 0

Получить виджет из определённой ячейки можно с помощью метода cellWidget (возвращает QWidget*, так что понадобится приведение типов):

QComboBox *comboBox = qobject_cast< QComboBox* >(ui->tableWidget->cellWidget(0, 0));
QString value = comboBox->currentText();
